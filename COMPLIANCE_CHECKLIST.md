# Peys Compliance Checklist

## Pre-Launch Requirements

### Business Registration

- [ ] CAC Certificate of Incorporation
- [ ] CAC Form C02 (List of Directors)
- [ ] Tax Identification Number (TIN)
- [ ] Bank Account Verification Letter
- [ ] NEPZA Registration (for international operations)

### Flutterwave Onboarding

- [ ] Flutterwave Business application submitted
- [ ] KYC Documentation:
  - [ ] Certificate of Incorporation
  - [ ] Memorandum & Articles of Association
  - [ ] Board Resolution authorizing account
  - [ ] Director ID (NIN or BVN)
  - [ ] Proof of Business Address
  - [ ] Bank Verification Letter
- [ ] Integration acceptance tests passed
- [ ] Service Agreement signed
- [ ] Production API credentials received

### Technical Security

- [ ] SSL/TLS Certificate (HTTPS)
- [ ] PCI DSS SAQ A-EP completed
- [ ] Attestation of Compliance (AoC) submitted
- [ ] API keys secured (environment variables)
- [ ] Fraud detection system implemented
- [ ] 2FA for withdrawal transactions
- [ ] Webhook signature verification

### CBN Compliance

- [ ] 2-Factor Authentication (2FA) system
- [ ] KYC process for all customers
- [ ] AML Policy documented
- [ ] Suspicious Transaction Reporting (STR) procedure
- [ ] Customer Due Diligence (CDD) process
- [ ] Record retention policy (6+ years)

### NDPR Compliance

- [ ] Privacy Policy published
- [ ] Cookie Consent mechanism
- [ ] Data subject rights handling
- [ ] Data breach notification procedure
- [ ] Data processing agreement with Flutterwave

## Daily Operations

### Transaction Monitoring

- [ ] Monitor transaction volume
- [ ] Review flagged/suspicious transactions
- [ ] Check settlement reconciliation
- [ ] Respond to customer inquiries within 24hrs
- [ ] Update transaction status in real-time

### Chargeback Management

- [ ] Review chargeback notifications daily
- [ ] Gather evidence within response deadline (7 days)
- [ ] Submit chargeback responses
- [ ] Track chargeback rate (keep below 1%)
- [ ] Maintain evidence records

### Compliance Checks

- [ ] Verify new customer KYC
- [ ] Update customer records as needed
- [ ] Review transaction patterns
- [ ] Check for sanctions list matches
- [ ] Update risk scores

## Incident Response

### Data Breach

1. [ ] Identify scope of breach
2. [ ] Contain the breach
3. [ ] Report to Flutterwave within 24 hours
4. [ ] Report to NDPR authority within 72 hours
5. [ ] Notify affected customers
6. [ ] Remediate vulnerabilities
7. [ ] Document lessons learned

### Fraud/Suspicious Activity

1. [ ] Flag transaction
2. [ ] Freeze affected accounts
3. [ ] Preserve evidence
4. [ ] Report to compliance officer
5. [ ] File STR if required
6. [ ] Coordinate with Flutterwave

### Service Disruption

1. [ ] Acknowledge issue
2. [ ] Communicate to customers
3. [ ] Work with Flutterwave support
4. [ ] Document incident
5. [ ] Implement fixes
6. [ ] Post-mortem review
